# ANDROID.md — Build the CheckSquare APK

> **Audience:** AI coding agents (Copilot, Claude, Cursor) and developers on a fresh
> Windows laptop. Tell the agent *"Read ANDROID.md and build me an APK"* and it
> has everything it needs — no Android Studio GUI required.

---

## 0. What this produces

A **debug APK** at:
```
apps\web\android\app\build\outputs\apk\debug\app-debug.apk   (~5–6 MB)
```

The APK is a thin Capacitor shell — a native Android app that opens a fullscreen
webview pointing at **<https://checksquare.pages.dev>**. Every web release pushed
to GitHub auto-deploys to Pages and the APK picks it up on next launch — no APK
reinstall needed for frontend changes.

For a **signed release APK** (for sharing widely, Play Store, etc.), see §7.

---

## 1. Ground rules for the agent

- **Do NOT** launch Android Studio. The GUI first-run wizard is interactive and
  can't be automated. Use the headless `sdkmanager` flow in §3.
- **Do NOT** commit `apps/web/android/` — it's regenerated by `npx cap add android`
  and is already in `.gitignore`.
- **Do NOT** commit any `*.apk`, `*.aab`, `*.keystore`, or `*.jks` file.
- **Do NOT** change `capacitor.config.json` `server.url` without asking the user
  — flipping to a bundled offline build breaks the auto-update workflow.
- If a step fails, **stop and report** the exact error. Do not guess fixes.
- Use absolute paths in PowerShell commands so the agent doesn't depend on cwd.

## 2. Prerequisites (one-time per laptop)

| Tool | Required version | Install command |
|---|---|---|
| Node.js | ≥ 20 | <https://nodejs.org> (LTS) |
| Microsoft OpenJDK | **21** (Capacitor 7 hard requirement) | `winget install --id Microsoft.OpenJDK.21 -e` |
| Android command-line tools | latest | §3 (auto) |
| Android platform-tools, build-tools 34.0.0, platforms;android-34 | — | §3 (auto) |

**Important — JDK version:** Capacitor 7's `capacitor-android` Gradle module
compiles with `sourceCompatibility 21`. JDK 17 will fail with
`error: invalid source release: 21`. JDK 11 fails earlier.

Verify after install:
```powershell
& "C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot\bin\java.exe" -version
# Expect: openjdk version "21.x.x"
```

## 3. Headless Android SDK install (idempotent)

Run this **once per laptop**. Downloads ~700 MB.

```powershell
# 3.1 — Download cmdline-tools
$sdkRoot = "$env:LOCALAPPDATA\Android\Sdk"
New-Item -ItemType Directory -Force -Path "$sdkRoot\cmdline-tools" | Out-Null

if (-not (Test-Path "$sdkRoot\cmdline-tools\latest\bin\sdkmanager.bat")) {
  $url = "https://dl.google.com/android/repository/commandlinetools-win-13114758_latest.zip"
  $tmp = "$env:TEMP\android-cmdline-tools.zip"
  Invoke-WebRequest -Uri $url -OutFile $tmp
  Expand-Archive -Path $tmp -DestinationPath "$sdkRoot\cmdline-tools" -Force
  if (Test-Path "$sdkRoot\cmdline-tools\latest") { Remove-Item "$sdkRoot\cmdline-tools\latest" -Recurse -Force }
  Rename-Item "$sdkRoot\cmdline-tools\cmdline-tools" "$sdkRoot\cmdline-tools\latest"
  Remove-Item $tmp
}

# 3.2 — Persist env vars (User scope, survives reboot)
$jdk21 = "C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot"   # adjust if version differs
[Environment]::SetEnvironmentVariable("ANDROID_HOME",     $sdkRoot, "User")
[Environment]::SetEnvironmentVariable("ANDROID_SDK_ROOT", $sdkRoot, "User")
[Environment]::SetEnvironmentVariable("JAVA_HOME",        $jdk21,   "User")

# Also set for current shell so the next commands work immediately
$env:ANDROID_HOME     = $sdkRoot
$env:ANDROID_SDK_ROOT = $sdkRoot
$env:JAVA_HOME        = $jdk21
$env:Path = "$jdk21\bin;$sdkRoot\cmdline-tools\latest\bin;$sdkRoot\platform-tools;" + $env:Path

# 3.3 — Accept all SDK licenses (pipe enough "y" answers for every prompt)
$sdkmgr = "$sdkRoot\cmdline-tools\latest\bin\sdkmanager.bat"
"y`n" * 20 | & $sdkmgr --licenses

# 3.4 — Install only what's needed for Capacitor 7
& $sdkmgr "platform-tools" "platforms;android-34" "build-tools;34.0.0"

# 3.5 — Verify
"platforms:     $(Test-Path "$sdkRoot\platforms\android-34")"
"build-tools:   $(Test-Path "$sdkRoot\build-tools\34.0.0")"
"platform-tools:$(Test-Path "$sdkRoot\platform-tools\adb.exe")"
```

All three should print `True`.

## 4. Project setup (one-time per clone)

After `git clone` + `npm install`:

```powershell
cd apps\web

# 4.1 — Build the web bundle (Capacitor copies this into the APK)
npm run build                # outputs to ../../dist/apps/web

# 4.2 — Scaffold the Android Gradle project from capacitor.config.json
#       (recreates apps/web/android/, which is gitignored)
npx cap add android

# 4.3 — Generate Android app icons from public/logo.svg
node scripts\make-icon.mjs                      # → resources/icon*.png
npx @capacitor/assets generate --android --assetPath resources
```

After this `apps\web\android\` exists and is ready to build.

---

## 5. Build the APK

Every time the web app changes:

```powershell
cd apps\web

# 5.1 — Rebuild web bundle
npm run build

# 5.2 — Copy fresh web assets + plugins into android/
npx cap sync android

# 5.3 — Compile APK with Gradle
cd android
$env:JAVA_HOME = "C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot"
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"
$env:Path = "$env:JAVA_HOME\bin;" + $env:Path
.\gradlew.bat assembleDebug
```

First build downloads Gradle 8.14 + dependencies (~5 min). Subsequent builds
take ~10 s thanks to the Gradle daemon.

**Output:** `apps\web\android\app\build\outputs\apk\debug\app-debug.apk`

## 6. Install on a phone

1. Copy `app-debug.apk` to the phone (USB, Google Drive, WhatsApp-to-self).
2. On phone: open the file → "Install from unknown sources" → Install.
3. Open **CheckSquare** → it loads `https://checksquare.pages.dev`.

> ⚠️ The APK needs internet to function. If login fails:
> 1. Confirm the Cloudflare tunnel is running on the laptop hosting PocketBase
>    (`.\start-tunnel.ps1`).
> 2. Confirm `VITE_PB_URL` in Cloudflare Pages env vars matches the current
>    `*.trycloudflare.com` URL.
> 3. Trigger a Pages redeploy.

## 7. Signed release APK (for distribution)

Debug APKs are signed with Android's debug keystore and show "untrusted source"
warnings. For sharing widely, build a **release APK** signed with your own key.

```powershell
# 7.1 — Generate a keystore (do this ONCE, keep it forever — losing it means
#       you can't push updates to anyone who installed the previous version)
$keystorePath = "C:\Users\$env:USERNAME\checksquare-release.keystore"
& "$env:JAVA_HOME\bin\keytool.exe" -genkey -v `
  -keystore $keystorePath `
  -alias checksquare `
  -keyalg RSA -keysize 2048 -validity 10000
# Answer the prompts. Remember the password.

# 7.2 — Reference it from android/app/build.gradle (edit signingConfigs block):
#       signingConfigs {
#         release {
#           storeFile file('C:/Users/.../checksquare-release.keystore')
#           storePassword 'YOUR_PASSWORD'
#           keyAlias 'checksquare'
#           keyPassword 'YOUR_PASSWORD'
#         }
#       }
#       buildTypes { release { signingConfig signingConfigs.release } }

# 7.3 — Build
cd apps\web\android
.\gradlew.bat assembleRelease
# Output: app\build\outputs\apk\release\app-release.apk
```

> 🔒 **Never commit the keystore or its password.** Both are already in
> `.gitignore` (`*.keystore`, `*.jks`). Store them in a password manager.

## 8. Troubleshooting decision tree

| Symptom | Likely cause | Fix |
|---|---|---|
| `error: invalid source release: 21` | `JAVA_HOME` points to JDK ≤ 17 | Set `JAVA_HOME` to JDK 21 (§2) |
| `Could not GET '...dl.google.com...' No such host is known` | DNS/network blip mid-download | Verify `Resolve-DnsName dl.google.com` works, rerun `gradlew assembleDebug` |
| `SDK location not found` | `ANDROID_HOME` not set in current shell | Reapply §3.2 env vars |
| `Failed to install the following Android SDK packages...licenses not accepted` | Skipped §3.3 | Run `sdkmanager --licenses` again, accept all |
| `BUG! exception in phase 'semantic analysis' in source unit '_BuildScript_'` | Gradle wrapper using wrong JDK | Run `gradlew --stop`, re-export `JAVA_HOME`, rebuild |
| App icon is still the default Android robot | Cached on device | Uninstall app on phone first, then reinstall |
| App opens but login fails | PocketBase tunnel down or `VITE_PB_URL` stale | §6 troubleshooting note |
| `npx cap add android` says "platform already exists" | `android/` from previous run | `Remove-Item apps\web\android -Recurse -Force` then re-run |

## 10. When the user asks "rebuild the APK"

Follow this exact loop:

1. Confirm prerequisites (§2). If JDK 21 or Android SDK missing → §3.
2. If `apps/web/android/` doesn't exist → §4 (one-time scaffold).
3. Run §5 build commands.
4. Report APK path + size:
   ```powershell
   $apk = "apps\web\android\app\build\outputs\apk\debug\app-debug.apk"
   $f = Get-Item $apk
   "APK: $($f.FullName)  ($([math]::Round($f.Length/1MB,2)) MB)  built $($f.LastWriteTime)"
   ```
5. Remind user: uninstall old version on phone before installing (avoids icon
   cache + signing-key mismatch issues).

---

## 11. When the user asks "change the app icon"

1. Replace `apps/web/public/logo.svg` (or supply a new 1024×1024 PNG and skip
   step 2).
2. Run:
   ```powershell
   cd apps\web
   node scripts\make-icon.mjs
   npx @capacitor/assets generate --android --assetPath resources
   ```
3. Run §5 to rebuild the APK.
4. Uninstall + reinstall on phone (Android caches launcher icons aggressively).

---

## 12. When the user asks "change the app name or package ID"

Edit [apps/web/capacitor.config.json](apps/web/capacitor.config.json):
- `appId` — reverse-DNS package name (e.g. `com.bhupathi.checksquare`).
  ⚠️ Changing this after publishing means existing installs can't upgrade.
- `appName` — display name shown under the launcher icon.

Then run `npx cap sync android` and rebuild (§5).

## 13. What the agent must NEVER do

- ❌ Open Android Studio (interactive wizard, can't be automated)
- ❌ Commit `apps/web/android/`, `*.apk`, `*.aab`, `*.keystore`, `*.jks`
- ❌ Use JDK 11, 17, or any version < 21 for the Gradle build
- ❌ Change `capacitor.config.json` `server.url` without explicit user request
- ❌ Run two builds in parallel (Gradle daemon will lock)
- ❌ Suggest installing the Android SDK via Android Studio "for convenience"
- ❌ Recommend bundling web assets into the APK unless user explicitly wants
  offline mode (it breaks the auto-update workflow)
